## 1. Позиционирование

### 1.1 Что такое Nuvio

Nuvio — open-source Telegram-бот для скачивания видео и аудио с YouTube, TikTok и Instagram. Бот предоставляет удобный интерфейс с inline-кнопками для выбора формата, автоматически кэширует отправленные файлы для мгновенной повторной доставки и включает аналитический веб-дашборд для мониторинга использования.

### 1.2 Для кого

| Аудитория | Задача |
|---|---|
| **Пользователи Telegram** | Быстро скачать видео/аудио в чате без сторонних сайтов и приложений |
| **Администраторы каналов** | Поднять собственный экземпляр бота для своей аудитории |
| **Разработчики** | Форкнуть, кастомизировать или расширить функционал под свои нужды |

### 1.3 Ключевое отличие

Nuvio — полностью самостоятельное решение: один `docker compose up` разворачивает и бот, и дашборд аналитики с авторизацией. Не зависит от внешних API-агрегаторов — использует yt-dlp напрямую с авто-обновлениями и множественными fallback-стратегиями.

### 1.4 Лицензия

Apache License 2.0 — свободное использование, модификация и распространение, включая коммерческое, с сохранением уведомления об авторстве. Без ограничений на количество пользователей, инстансов или платформ.

## 3. Пользовательские сценарии

### 3.1 Скачивание видео (основной сценарий)

```
Пользователь отправляет ссылку →
  Бот определяет платформу →
    Получает метаданные (название, автор, длительность) →
      Показывает меню с кнопками выбора формата →
        Пользователь нажимает кнопку →
          Бот проверяет кэш:
            HIT → мгновенная отправка из Telegram CDN
            MISS → скачивание → отправка → сохранение в кэш
```

### 3.2 Меню для YouTube

- **📹 Скачать видео для ТГ** — лучшее качество, умещающееся в 50MB
- **🎵 Скачать только звук** — нативный m4a или конвертация
- **➕ Дополнительно** — расширенное меню:
  - До 3 комбинированных форматов с указанием разрешения
  - До 3 форматов «только видео» (без звука)
  - До 2 форматов «только аудио»
  - MP3 минимального размера
  - Лучшее качество (bestvideo + bestaudio)
  - Лучшее аудио
  - Субтитры (SRT)

### 3.3 Меню для TikTok / Instagram

- **Скачать видео** — полное видео с аудио
- **Только аудио (MP3)** — извлечение аудиодорожки

### 3.4 Обработка больших файлов (>50MB)

```
Файл >50MB →
  Gokapi настроен?
    Да → загрузка на Gokapi → ссылка пользователю (1 скачивание, 7 дней)
    Нет → сообщение "выберите меньший формат"
```

Кнопка «Лучшее качество» предупреждает `(может не влезть в ТГ)` если Gokapi не настроен.

## 4. Системная архитектура

### 4.1 Стек технологий

| Компонент | Технология |
|---|---|
| **Язык** | Python 3.13+ |
| **Telegram API** | python-telegram-bot 22+ (async) |
| **Загрузка видео** | yt-dlp (rolling-release, nightly) |
| **Медиа-обработка** | FFmpeg (извлечение аудио, конвертация, мерж) |
| **WebUI** | FastAPI + Jinja2 + Uvicorn |
| **База данных** | SQLite (WAL mode) — кэш + аналитика |
| **HTTP-клиент** | httpx (async, для Gokapi) |
| **CI/CD** | GitHub Actions (lint, test, Docker → GHCR) |
| **Контейнеризация** | Docker (python:3.13-slim + FFmpeg) |

### 4.2 Архитектурные паттерны

- **Async event-loop** — `asyncio` для обработки Telegram-обновлений
- **ThreadPoolExecutor** — блокирующие операции (yt-dlp) выполняются в пуле потоков (`DOWNLOAD_WORKERS=8`)
- **SQLite WAL** — конкурентное чтение/запись без блокировок для кэша и аналитики
- **Session tokens** — каждый запрос скачивания получает уникальный 8-символьный hex-токен; до 5 параллельных сессий на пользователя
- **Graceful shutdown** — корректное завершение по SIGINT/SIGTERM с очисткой временных файлов

### 4.3 Слои кэширования

```
┌─────────────────────────────────────────────┐
│              Telegram CDN                    │
│   Мгновенная доставка по file_id (0 сек)    │
│   TTL: 90 дней, автоочистка ежедневно       │
└─────────────────────────────────────────────┘
                    ↑ cache hit
                    │
┌─────────────────────────────────────────────┐
│            SQLite video_cache                │
│   PK: (url, format_id)                       │
│   Хранит: file_id, platform, size, duration  │
│   Индексы: url, file_id, platform, cached_at │
└─────────────────────────────────────────────┘
                    ↑ cache miss
                    │
┌─────────────────────────────────────────────┐
│           yt-dlp download                    │
│   Скачивание → FFmpeg → Telegram upload      │
│   Результат: file_id → сохранение в кэш     │
└─────────────────────────────────────────────┘
```

### 4.4 Потоки обработки

**YouTube:**
```
URL → get_video_info (cookies-first) → формат-меню → выбор →
  → проверка кэша → [download_content | download_audio_native | download_audio] →
  → FFmpeg (при необходимости) → проверка размера →
  → [Telegram upload | Gokapi] → сохранение file_id в кэш
```

**TikTok:**
```
URL → get_tiktok_info (3 API-конфига, exponential backoff) →
  → формат-меню → выбор → проверка кэша →
  → download (cookies rotation) → проверка размера →
  → [Telegram upload | Gokapi] → сохранение file_id в кэш
```

**Instagram:**
```
URL → валидация (отсечение Stories, Audio) →
  → get_instagram_info (без cookies → с cookies при ограничении) →
  → формат-меню → выбор → проверка кэша →
  → download (playlist handling) → проверка размера →
  → [Telegram upload | Gokapi] → сохранение file_id в кэш
```

### 4.5 Стратегии отказоустойчивости

| Уровень | Стратегия |
|---|---|
| **yt-dlp API** | Cookies-first → без cookies → CLI fallback |
| **TikTok API** | 3 API-хоста с автоматической ротацией |
| **Instagram** | Без cookies → с cookies при rate-limit |
| **Retry** | Exponential backoff (1s, 2s, 4s) для сетевых ошибок |
| **Telegram upload** | 3 попытки с exponential backoff |
| **Формат** | Фильтр по размеру → fallback на меньший формат |
| **Кэш** | При устаревании file_id — автоудаление и повторное скачивание |

## 5. WebUI — Аналитический дашборд

### 5.1 Страницы

| Страница | Описание |
|---|---|
| **Логин** | Авторизация по логину/паролю с fail2ban-защитой |
| **Дашборд** | KPI-карточки, графики, когортный анализ |
| **Список пользователей** | Таблица со всеми пользователями, поиск, сортировка |
| **Детали пользователя** | Профиль, статистика, скачивания по платформам, лог событий |

### 5.2 KPI-метрики дашборда

**Пользователи:**
- Всего пользователей
- Новые сегодня / за 7 дней / за 30 дней
- Активные сегодня / за 7 дней / за 30 дней
- Всего скачиваний

**Удержание и отток:**
- Retention D3, D7, D30 (% вернувшихся пользователей)
- Churn 30д (% оттока за 30 дней)

**Продуктовые метрики:**
- Среднее скачиваний на пользователя
- Повторные пользователи (% с >1 скачиванием)
- Вовлечённость DAU/MAU (stickiness ratio)

### 5.3 Графики

- **Скачиваний в день** — bar chart (30 дней)
- **Новых пользователей в день** — line chart с fill (30 дней)
- **Вовлечённость DAU/MAU** — dual-axis chart (DAU line + stickiness %)
- **Использование платформ** — doughnut chart + horizontal bars (охват пользователей)

### 5.4 Когортный анализ

Таблица удержания по недельным когортам (W0–W8) с цветовым кодированием интенсивности.

### 5.5 Безопасность WebUI

| Мера | Реализация |
|---|---|
| **Fail2ban** | N попыток → блокировка IP на M минут (настраивается) |
| **Timing-safe сравнение** | `hmac.compare_digest` для логина/пароля |
| **HMAC-сессии** | `SessionMiddleware` с подписью через `WEB_SECRET_KEY` |
| **Jinja2 autoescape** | Защита от XSS |
| **Swagger отключён** | `docs_url=None, redoc_url=None` |
| **Длина ввода** | Логин и пароль обрезаются до 128 символов |

## 6. Администрирование

### 6.1 Команды бота

| Команда | Описание |
|---|---|
| `/admin` | Панель управления cookies (загрузка, проверка здоровья) |
| `/cache_stats` | Статистика кэша: кол-во записей, разбивка по платформам |
| `/cleanup_cache` | Ручная очистка записей старше 90 дней |
| `/search_cache <запрос>` | Поиск в кэше по названию видео |

### 6.2 Управление cookies

Через `/admin` администратор может:

1. **Загрузить** cookies для YouTube, TikTok, Instagram (файл Netscape .txt ≤1MB)
2. **Проверить здоровье** — валидация каждого файла cookies с детальным статусом:
   - ✅ Valid — рабочие cookies
   - ❌ Expired / Stale / Invalid Format
   - ⚠️ Rate Limited / Probe Failed

### 6.3 Автоматические задачи

| Задача | Расписание | Описание |
|---|---|---|
| Очистка кэша | Каждые 24 часа | Удаление записей старше 90 дней |
| VACUUM базы данных | Каждые 7 дней | Оптимизация размера SQLite-файлов |
| Обновление yt-dlp | При старте | `pip install --upgrade` из nightly-канала |

### 6.4 Crash-репорты

При необработанном исключении бот автоматически отправляет crash-репорт всем администраторам:

- Код ошибки (формат: `PREFIX-CATEGORY-RANDOM`, например `YT-TIMEOUT-A3F2B1`)
- Платформа, стадия, URL, session ID
- Тип исключения и traceback
- Статус cookies (автопроверка)

## 7. Защита и ограничения

### 7.1 Антиспам

- **Лимит:** 4 запроса за 5 секунд → cooldown 10 секунд
- **Область:** Per-user, применяется к скачиваниям (не к навигации)
- **Параллельные сессии:** Максимум 5 активных меню на пользователя

### 7.2 Ограничения контента

| Ограничение | Значение |
|---|---|
| Максимальная длительность видео | 3 часа |
| Лимит файла Telegram | 50 MB |
| Файлы >50MB | Gokapi (если настроен) |
| Instagram Stories | Не поддерживаются |
| Instagram Audio posts | Не поддерживаются |

### 7.3 Безопасность данных

- SQL-запросы — только параметризованные (`?`)
- Ошибки — не раскрывают внутреннюю структуру или стектрейсы пользователю
- Секреты — хранятся в `.secrets/`, не попадают в git
- Cookies — загружаются только админами, валидируются перед использованием

## 8. Развёртывание

### 8.1 Docker (рекомендуется)

```bash
cp .env.example .env
# Заполнить TELEGRAM_TOKEN, ADMIN_IDS, WEB_PASSWORD
docker compose up --build
```

Два контейнера:
- `nuvio-bot` — Telegram-бот
- `nuvio-web` — WebUI-дашборд (порт `WEB_PORT`, по умолчанию 8080)

Общий volume `bot-data` для SQLite-баз данных.

### 8.2 Bare metal

```bash
pip install -r requirements.txt
# Настроить .secrets/.env
python main.py           # бот
python -m web.app        # дашборд (отдельный процесс)
```

**Системные зависимости:** Python 3.13+, FFmpeg, git.

### 8.3 Production (GHCR)

При пуше тега `v*` GitHub Actions автоматически:
1. Прогоняет тесты
2. Генерирует changelog
3. Создаёт GitHub Release
4. Собирает Docker-образ → GHCR с тегами `latest`, `X.Y.Z`, `X.Y`, `X`

```bash
TAG=1.0.0 docker compose -f docker-compose.prod.yml up -d
```

### 8.4 CI/CD Pipeline

```
Push/PR → CI:
  ├── Lint (ruff)
  ├── Tests (Python 3.13 + 3.14)
  └── Docker build check

Tag v* → Release:
  ├── Tests
  ├── Changelog generation
  ├── GitHub Release
  └── Docker → GHCR
```
